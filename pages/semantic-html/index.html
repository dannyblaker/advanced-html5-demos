Document Structure with Semantic Elements

HTML5 provides semantic elements that describe the meaning of content, not just its appearance.

<header> - Site header, page header, or section header

Contains headings, logos, navigation, introductory content

<nav> - Navigation links

Primary navigation, breadcrumbs, pagination

<main> - Main content area

The primary content of the document (only one per page)

<section> - Thematic grouping of content
<article> - Independent, self-contained content
<aside> - Sidebar content related to main content
<aside> - Sidebar content

Related links, advertisements, author info, etc.

<footer> - Footer information

Copyright, contact info, related links

<!-- Semantic HTML5 Document Structure -->
<header>
    <h1>Site Title</h1>
    <nav>
        <ul><li><a href="/">Home</a></li></ul>
    </nav>
</header>

<main>
    <article>
        <header>
            <h1>Article Title</h1>
            <time datetime="2025-10-07">October 7, 2025</time>
        </header>
        <p>Article content...</p>
        <footer>
            <p>By <address>Author Name</address></p>
        </footer>
    </article>
    
    <aside>
        <h2>Related Articles</h2>
        <!-- Related content -->
    </aside>
</main>

<footer>
    <p>© 2025 Example Site</p>
</footer>

Text-level Semantic Elements

Demonstrations

<strong> - Important text (semantic emphasis)

<em> - Emphasized text (stress emphasis)

<mark> - Highlighted text (relevance)

<small> - Fine print, disclaimers

<del> - Deleted text (removed content)

<ins> - Inserted text (added content)

<sub> - H2O (subscript)

<sup> - E=mc2 (superscript)

<code> - const variable = 'value';

<kbd> - Press Ctrl+C to copy

<samp> - Command executed successfully

<var> - x = y + 2

"The HTML5 specification defines the meaning of each element, which helps browsers, search engines, and assistive technologies understand your content."

— W3C HTML5.2 Specification

Code Examples

<!-- Emphasis and importance -->
<p>This is <strong>very important</strong>!</p>
<p>You <em>really</em> should know this.</p>

<!-- Citations and quotes -->
<blockquote cite="source-url">
    <p>Quote text here</p>
    <footer>— <cite>Source</cite></footer>
</blockquote>

Document Outline and Heading Structure

Proper Heading Hierarchy

H1 - Page Title

H2 - Section Title
H3 - Subsection Title
H4 - Sub-subsection

Benefits:

  • Screen readers can navigate by headings
  • Search engines understand content hierarchy
  • Automatic table of contents generation
  • Better document structure

Section-based Outline

<!-- Each section resets heading level -->
<section>
    <h1>Section Title</h1>
    <section>
        <h1>Subsection Title</h1>
        <p>Content...</p>
    </section>
</section>

<article>
    <h1>Article Title</h1>
    <section>
        <h1>Article Section</h1>
        <p>Content...</p>
    </section>
</article>
